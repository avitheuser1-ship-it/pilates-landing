# לומרה — קובץ ייבוא קמפיינים ל-Google Ads Editor

קובץ: `Lumera_Ads_Bulksheet.csv` (57 שורות, UTF-8 עם BOM — נקרא בעברית תקין ב-Editor)

## מה בקובץ
| קמפיין | שם קבוע | תוכן |
|---|---|---|
| קמפיין חיפוש | ✓ | 2 קבוצות (צורך-בריאות / גישה-שרון), 14 מילות מפתח (ללא רחבה וללא ביטוי בודד), 2 מודעות RSA, תוספים: Sitelinks + Callouts + Structured Snippets + מילים שליליות |
| קמפיין מותג | ✓ | קבוצה 1, 5 מילות מפתח כולל לומרה/LUMERA, **DKI ({KeyWord:לומרה}) בכל הכותרות והתיאורים** |
| קמפיין GDN | ✓ | 2 קבוצות (לא-המירו / אפסייל), 2 מודעות דיספליי — מכוונות לקהלי הרימרקטינג |
| קמפיין וידאו | ✓ | קבוצה 1, מודעת וידאו (מפנה לסרט לדוגמה — להחליף ל-4 ערוצים + 2 סרטונים) |

**כל הקמפיינים מסומנים Paused** — לא יוצאים כסף עד שתפעיל ידנית.

## איך מייבאים (בלי גישה לחשבון)
1. הורד Google Ads Editor (https://ads.google.com/intl/he/editor/) והתחבר לחשבון שלך.
2. `File` → `Import` → `From CSV file…` → בחר את `Lumera_Ads_Bulksheet.csv`.
3. בודק שהכל נטען ללא שגיאות (העמודה Status = Paused).
4. `Post` (פרסם) — זה יוצר את הקמפיינים **מושהים** בחשבון.

## מה לעשות ידנית (לא נכנס ל-CSV)
### א. קהלי רימרקטינג (אופטימיזציה → קהלים → + קהל רימרקטינג אתר)
- `גולשים למוצר הראשון שלא המירו` — כלל: דפים שכוללים `product1.html` MINUS קהל ההמרה.
- `ביצעו המרה למוצר הראשון` — כלל: דפים שכוללים `thank-you.html`.
- לאחר מכן: בקמפיין GDN → קבוצה "לא-המירו" ← קהל ראשון; קבוצה "אפסייל" ← קהל שני.

### ב. קוד המרה (כך שלומרה ייספר המרה במילוי טופס)
1. בחשבון: `כלים וכללי הגדרות` → `מקודות נתונים` → `פעולות המרה` → `+ המרה באתר`.
2. שם: `מילוי טופס לומרה` | קטגוריה: רישום | ערך: לבחירה (למשל 10).
3. העתק את קוד ה-GTAG ושלח לי (או הדבק בדפים תחת `AW-XXXXXXXXX`) → אני דוחף ל-GitHub Pages.
   - לחלופין: תשלח לי את ה-Customer ID + הלייבל, ואני מחבר.

### ג. קמפיין וידאו — להשלים טרגוט
בקובץ יש מודעה אחת עם סרט לדוגמה. בחשבון, בקמפיין הוידאו, הוסף:
- 4 ערוצי YouTube רלוונטיים (פילאטיס/כושר/בריאות)
- 2 סרטונים (ניתן להשתמש בכל סרט ציבורי)
